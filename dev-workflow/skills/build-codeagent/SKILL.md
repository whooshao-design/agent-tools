# build-codeagent

## 定位

用于作为 `dev-build-change` 和 `dev-auto-loop` 的执行后端增强，统一承接 codeagent 类调用。

它负责约束多后端调用方式、超时、恢复、并行和失败处理策略，不替代开发主入口本身的代码判断。

默认不作为通用主入口；仅在主开发流程判断需要子代理、并行或长任务执行时使用。

## When to Use

适合以下场景：
- 当前改动较大，单次上下文过重，适合交给子代理执行
- 需要在 Codex、Claude 或其他已接入后端之间切换执行
- 需要并行处理相互独立的实现任务
- 需要为长耗时任务设置统一超时、恢复和失败回退策略

不适合以下场景：
- 小改动可直接在当前主流程完成
- 需求或方案尚未稳定，不适合放大执行
- 只是做一般搜索、问答或轻量修改，不需要 codeagent

## 核心职责

统一处理以下事项：
- 后端选择：按任务性质选择最合适的 codeagent 后端
- 安全约束：注入工作区边界和禁止越界删除等规则
- 超时策略：长任务显式设置超时，避免默认超时中断
- 恢复策略：保留会话恢复方式，避免中断后完全重来
- 并行策略：仅对独立任务并行，依赖任务保持串行

## 使用原则

- codeagent 只是执行增强，不负责定义需求和方案
- 调用前先明确目标、边界、输入文件和完成标准
- 默认带安全约束，尤其限制删除和工作区外操作
- 长任务必须显式设置超时；恢复时必须保持后端一致
- 并行前先确认任务之间没有文件级冲突或顺序依赖

## 输出要求

输出至少说明：
- 选择了哪个后端以及原因
- 是否使用了并行或恢复机制
- 子任务完成状态与失败点
- 哪些结果可直接吸收回主开发流程

## 交接建议

- 实现结果回交 `dev-build-change` 汇总落地
- 若用于自动循环，由 `dev-auto-loop` 决定是否继续下一阶段
